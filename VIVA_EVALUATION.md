### 4. HASHING & DIGITAL SIGNATURE (3 Marks) ✅

#### 4.1 Hashing with Salt (1.5 Marks) ✅

**Implementation**: Password hashing with per-user random salt

| Property | Value |
|----------|-------|
| Algorithm | scrypt (memory-hard function) |
| Parameters | n=32768, r=8, p=1 |
| Salt | Random per user (unique) |
| Output Format | `scrypt:32768:8:1$<salt>$<hash>` |

```python
# Same password, different users = different hashes
User 1: "password123" → "scrypt:32768:8:1$aB3xY$1f2e..."
User 2: "password123" → "scrypt:32768:8:1$zK9mW$9a8b..."
```

**Protection Against**:
- ✅ Rainbow table attacks (unique salt per user)
- ✅ Brute force (scrypt is memory-hard, resists GPU/ASIC attacks)
- ✅ Database breaches (no plaintext storage)

**Code Location**: [routes/auth_routes.py](Backend/routes/auth_routes.py) - Registration & Login

#### 4.2 Digital Signature using Hash (1.5 Marks) ✅

**Implementation**: RSA-2048 digital signature with SHA-256 hash

**What Gets Signed**:
```
message = f"{user_id}|{quiz_id}|{score}|{timestamp}"
# Example: "123|45|8.5|2026-01-12T14:27:17.257170"
```

**Process**:
```
1. Create message string
2. SHA-256 hash (using hashlib)
3. RSA-2048 sign with private key (PSS padding)
4. Base64 encode for storage
5. Store in Score.digital_signature
```

```python
import hashlib
from cryptography.hazmat.primitives.asymmetric import padding

# Hash with SHA-256
hash_obj = hashlib.sha256(message.encode('utf-8'))
digest = hash_obj.digest()

# Sign with RSA private key
signature = private_key.sign(
    digest,
    padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
    hashes.SHA256()
)
```

**Verification Endpoint**: `GET /api/user/verify-signature/<score_id>`

**Security Benefits**:
- ✅ **Integrity**: Cannot modify score without invalidating signature
- ✅ **Authenticity**: Proves result came from trusted server
- ✅ **Non-repudiation**: Cryptographic proof of quiz completion

**Code Location**: [crypto_utils.py#L145-L250](Backend/crypto_utils.py)

### 5. ENCODING TECHNIQUES (3 Marks) ✅

#### 5.1 Encoding & Decoding Implementation (1 Mark) ✅

**Two Encoding Techniques Implemented**:

**1. Base64 Encoding** - Quiz Result Verification Tokens
| Property | Value |
|----------|-------|
| Tool | Python `base64` module |
| Character Set | A-Z, a-z, 0-9, +, /, = |
| Purpose | Compact representation of result data |
| Storage | `Score.verification_token` field |

```python
# Encode
verification_token = base64.b64encode(
    f"{user_id}|{quiz_id}|{score}|{timestamp}".encode()
).decode('utf-8')
# Result: "MTIzfDQ1fDguNXwyMDI2LTAxLTEyVDE0OjU4OjQ2"

# Decode
decoded = base64.b64decode(token).decode('utf-8')
# Result: "123|45|8.5|2026-01-12T14:58:46"
```

**2. Hexadecimal (Base16) Encoding** - Quiz Integrity Hashes
| Property | Value |
|----------|-------|
| Tool | Python `bytes.hex()` / `bytes.fromhex()` |
| Character Set | 0-9, a-f |
| Purpose | Human-readable integrity hashes |
| Storage | `Quiz.integrity_hash` field |

```python
# Generate hex hash
integrity_hex = hashlib.sha256(quiz_content.encode()).hexdigest()
# Result: "3bad936a49c8a12c5e631425657556371b3f318b..."
```

**Code Location**: [crypto_utils.py#L300-L400](Backend/crypto_utils.py)

#### 5.2 Security Levels & Risks (Theory) (1 Mark)

| Encoding | Security Level | Risk |
|----------|----------------|------|
| **Base64** | LOW - Obfuscation only | Easily reversible, NOT encryption |
| **Hex** | LOW - Format conversion | Just representation change |
| **AES-256** | HIGH - True encryption | Requires key to decrypt |
| **RSA-2048** | HIGH - Asymmetric | Private key must be protected |

**Key Insight**: Encoding ≠ Encryption
- Encoding: Transforms data format (reversible by anyone)
- Encryption: Protects data (requires key to reverse)

#### 5.3 Possible Attacks (Theory) (1 Mark)

| Attack | Description | Mitigation in App |
|--------|-------------|-------------------|
| **Rainbow Table** | Pre-computed hash lookup | Random salt per user |
| **Brute Force** | Try all combinations | scrypt memory-hard, rate limiting |
| **SQL Injection** | Malicious SQL in input | SQLAlchemy ORM, parameterized queries |
| **JWT Tampering** | Modify token claims | Signed with secret key |
| **Replay Attack** | Reuse captured OTP | 5-minute expiry, single-use |
| **Man-in-the-Middle** | Intercept communication | HTTPS (production), JWT tokens |
| **Database Breach** | Steal database | Passwords hashed, answers encrypted |
| **Signature Forgery** | Fake quiz results | RSA-2048 private key required |
